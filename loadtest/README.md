# 부하 테스트 (동시성 검증)

## 목적
타임딜 상황에서 Redis Lua 기반 원자적 재고 차감 게이트가
동시 요청 폭주 시에도 초과판매(oversell)를 막는지 검증한다.

## 시나리오
- 재고 100개 상품에 동시 1,000요청 (k6, VU 1000)
- 재고 차감 게이트만 격리하여 측정 (로그인/장바구니/배송 제외, 동시성 코어만 순수 측정)

## 실행 순서
1. Redis 재고 초기화: redis-cli SET product:stock:1 100
2. 부하 테스트 실행: k6 run flash-sale.js
3. 최종 재고 확인: redis-cli GET product:stock:1  (기대값 0)

## 결과
- order_success: 정확히 100건 (재고 수량과 일치)
- 최종 재고: 0 (음수 없음 -> 초과판매 0건)
- 재고 정합성: 동시 1,000요청에서도 100% 보장됨

## 발견
- 재고 로직은 완벽히 동작했으나(정합성 100%), 동시 1,000요청 중 약 900건이
  타임아웃 -> 병목이 재고 로직이 아니라 애플리케이션 커넥션 수용 한계
  (EC2 단일 인스턴스)에 있음을 확인. 스케일 아웃/커넥션 풀 튜닝을 다음 과제로 도출.
